## Versioning

`flutter_blue_ultra` follows the legacy `flutter_blue_plus` 1.x API and uses traditional versioning for the app-facing package:

`BIG.MEDIUM.SMALL`

- BIG: Significant overhauls, e.g. 1.0.0 → 2.0.0
- MEDIUM: Moderate improvements, breaking changes possible, e.g. 1.0.0 → 1.1.0
- SMALL: Fixes and refinements, e.g. 1.0.0 → 1.0.1

Platform implementations (`android`, `darwin`, `linux`, `platform_interface`, `web`) use Semantic Versioning (`MAJOR.MINOR.PATCH`).
